# Vision: 시황 보고서에서 자율 자산배분 시스템으로

> 2026-04-12 작성

## 한 문장 비전

**시장 데이터를 수집·해석하고 자산배분을 결정하는 전 과정을, 사람이 읽는 보고서에서 시작해 Multi-Agent가 자율적으로 운용하는 시스템으로 진화시킨다.**

---

## 진화의 3단계

```
Stage 1                    Stage 2                      Stage 3
사람을 돕는 도구     →     사람과 협업하는 에이전트  →    자율 운용 에이전트 팀
───────────────────────────────────────────────────────────────────────
시황 보고서              자산배분 View +                Multi-Agent가
Data + Story             모델 포트폴리오 제시           스스로 판단·실행
                         사람이 승인·실행               사람은 감독·개입
```

---

## Stage 1 — 사람을 돕는 도구 (현재)

시장 데이터를 자동 수집하고, 사람이 읽기 좋은 형태로 정리해준다.

- 56개 글로벌 지표 일일 수집
- Data Dashboard + Claude가 작성한 Market Story
- 주간/월간 집계 보고서
- **의사결정 주체**: 사람. 시스템은 정보 제공만

이 단계의 가치: 매일 반복되는 데이터 수집·정리 작업을 자동화해서 사람이 해석과 판단에 집중할 수 있게 한다.

---

## Stage 2 — 사람과 협업하는 에이전트 (ROADMAP Phase 0-6)

시스템이 데이터를 해석하고 자산배분 의견까지 제시한다. 최종 결정은 사람이 한다.

- 자산군별 OW/N/UW View 자동 생성
- 룰 기반 모델 포트폴리오 산출 (60/40, Risk Parity, TAA)
- 리밸런싱 시그널 + 백테스트 검증
- PM/PB 이중 페르소나 해설
- **의사결정 주체**: 사람. 시스템은 분석·추천·근거 제시

이 단계의 핵심 원칙:
1. **투명성** — 왜 이 View인지, 어떤 시그널이 작동했는지 항상 설명
2. **검증 가능성** — 백테스트로 과거 성과를 보여줘서 신뢰 확보
3. **사람의 판단 우선** — 시스템 추천을 무시할 수 있는 구조

---

## Stage 3 — 자율 운용 Multi-Agent 시스템 (최종 목표)

여러 전문 에이전트가 각자의 역할로 정보를 수집·분석하고, 협업을 통해 자산배분을 스스로 결정·실행한다. 사람은 감독자로서 위험 한도와 운용 철학을 설정하고, 예외 상황에만 개입한다.

### Agent 구성

```
┌─────────────────────────────────────────────────────┐
│                  Supervisor Agent                    │
│         운용 철학 · 위험 한도 · 최종 승인             │
└──────────┬──────────────┬──────────────┬────────────┘
           │              │              │
    ┌──────▼──────┐ ┌─────▼──────┐ ┌────▼───────┐
    │ Macro Agent │ │ Market     │ │ Risk       │
    │             │ │ Sentiment  │ │ Agent      │
    │ 매크로 경제  │ │ Agent      │ │            │
    │ 데이터 분석  │ │            │ │ 포트폴리오  │
    │ FRED, 금리,  │ │ 뉴스, 심리, │ │ 위험 모니터 │
    │ 고용, 물가   │ │ 포지셔닝   │ │ VaR, MDD,  │
    │             │ │            │ │ 상관관계    │
    └──────┬──────┘ └─────┬──────┘ └────┬───────┘
           │              │              │
    ┌──────▼──────┐ ┌─────▼──────┐ ┌────▼───────┐
    │ Sector      │ │ Quant      │ │ Execution  │
    │ Agent       │ │ Agent      │ │ Agent      │
    │             │ │            │ │            │
    │ 섹터·테마    │ │ 모멘텀,     │ │ 주문 실행,  │
    │ 로테이션     │ │ 밸류에이션,  │ │ 리밸런싱,   │
    │ 분석        │ │ 시그널 통합  │ │ 체결 관리   │
    └─────────────┘ └────────────┘ └────────────┘
```

### 각 Agent의 역할

| Agent | 입력 | 출력 | 판단 |
|-------|------|------|------|
| **Macro Agent** | FRED, ECOS, 중앙은행 발표, 경제 캘린더 | 매크로 국면 판단 (확장/둔화/침체/회복) | 금리·성장·물가 사이클 위치 |
| **Market Sentiment Agent** | 뉴스 헤드라인, VIX, put/call ratio, fund flow, positioning | 시장 심리 점수 (공포/탐욕) | 역발상 시그널, 이벤트 리스크 |
| **Risk Agent** | 포트폴리오 현황, 상관행렬, 변동성 | 위험 예산 배분, 한도 위반 경고 | 위험 조정 비중 제안 |
| **Sector Agent** | 섹터 ETF 가격, 실적 캘린더, 테마 뉴스 | 섹터 OW/UW, 테마 노출 | 업종 로테이션 타이밍 |
| **Quant Agent** | 전체 시계열, 시그널 스코어 | 자산별 종합 점수, 목표 비중 | 시그널 가중·통합 |
| **Execution Agent** | 목표 비중, 현재 포지션, 시장 상태 | 주문 생성·체결·보고 | 실행 타이밍, 분할 매매 |
| **Supervisor Agent** | 모든 Agent의 출력, 사람이 설정한 제약 | 최종 자산배분 결정, 사람에게 보고 | 에이전트 간 충돌 조정, 이례 상황 판단 |

### 의사결정 흐름

```
매일 08:00 KST

1. 데이터 수집
   Macro Agent     → 매크로 데이터 수집·해석 → 국면 판단
   Sentiment Agent → 뉴스·심리 데이터 수집   → 심리 점수
   Sector Agent    → 섹터·테마 데이터 수집   → 로테이션 의견

2. 분석·통합
   Quant Agent     → 모든 시그널 수신 → 자산별 종합 점수 → 목표 비중 초안
   Risk Agent      → 목표 비중 검증 → 위험 한도 내 조정

3. 결정
   Supervisor Agent → 최종 비중 확정
                    → 직전 비중과 비교 → 리밸런싱 필요 여부 판단
                    → 변경 사유 기록

4. 실행
   Execution Agent → 리밸런싱 주문 생성·실행
                   → 체결 결과 기록

5. 보고
   Supervisor Agent → 일일 운용 보고서 생성
                    → 사람(감독자)에게 요약 전달
```

### 사람의 역할 변화

| 항목 | Stage 2 (협업) | Stage 3 (자율) |
|------|---------------|---------------|
| 매크로 판단 | 사람이 직접 | Agent가 판단, 사람은 검토 |
| 비중 결정 | 사람이 최종 결정 | Agent가 결정, 사람은 한도 설정 |
| 리밸런싱 실행 | 사람이 승인 후 실행 | Agent가 자동 실행 (한도 내) |
| 예외 상황 | 사람이 직접 대응 | Agent가 1차 대응, 사람에게 보고 |
| 운용 철학 | 사람이 매일 반영 | 사람이 분기별 재설정 |

### Stage 3 진입 조건

Stage 2에서 충분한 신뢰가 쌓인 후에만 전환한다:

1. **Paper trading 6개월 이상** — 모델 포트폴리오의 실시간 성과 검증
2. **View 적중률 60% 이상** — OW 자산이 UW 자산을 outperform
3. **MDD 관리 입증** — drawdown 임계 도달 시 자동 방어 작동 확인
4. **Agent 간 합의 프로토콜 안정화** — 충돌 시 해소 로직 검증
5. **사람의 판단** — 시스템을 신뢰할 수 있다는 운영자의 확신

---

## 기술 원칙

이 세 단계를 관통하는 설계 원칙:

1. **점진적 자율성** — 한 번에 자율 시스템을 만들지 않는다. Stage 1의 데이터 파이프라인 위에 Stage 2의 분석 레이어를, 그 위에 Stage 3의 에이전트를 올린다. 각 단계가 이전 단계의 코드와 데이터를 그대로 활용한다.

2. **설명 가능성** — 모든 단계에서 "왜 이 결정인가"를 추적할 수 있어야 한다. Agent가 자율적으로 결정하더라도 근거 로그는 반드시 남긴다.

3. **사람이 언제든 개입** — Stage 3에서도 사람은 Agent를 멈추거나, 비중을 직접 오버라이드하거나, 운용 철학을 변경할 수 있다. 자율성은 위임이지 포기가 아니다.

4. **실패에 안전하게** — Agent가 잘못된 판단을 해도 손실이 제한되도록 위험 한도(position limit, drawdown limit, turnover cap)를 하드코딩한다.

5. **단일 데이터 소스** — `history/market_data.csv`와 Snowflake가 모든 단계의 공통 데이터 기반. 새로운 Agent가 추가돼도 같은 데이터를 읽는다.
